D7 续：rcc_sysclk_init —— 把芯片从 8MHz 拉到 72MHz

为什么要做这个？

STM32F103 上电默认跑内部 HSI 8MHz，但芯片能跑 72MHz——快了 9 倍。上午你写的 rcc_enable(GPIOx) 只管开外设时钟，不管系统主频。下午加上 rcc_sysclk_init()，rcc 模块才完整。

🔑 rcc.c 现在两个函数的分工：
rcc_sysclk_init() → 配系统主频（全局，调用一次）
rcc_enable(GPIOx) → 开外设时钟（局部，用哪个开哪个）

时钟路径图

HSE 8MHz（外部晶振） ↓ PLLSRC = 1（选 HSE） ↓ PLLXTPRE = 0（进 PLL 前不分频） ↓ PLLMUL = 0111（×9） 72MHz（PLL 输出） ↓ SW = 10（系统时钟选 PLL） SYSCLK = 72MHz ├→ AHB ÷1 = 72MHz ├→ APB2 ÷1 = 72MHz（GPIOA/B/C、AFIO、SPI1、USART1 在这） └→ APB1 ÷2 = 36MHz（USART2/3、SPI2、I2C 在这）

完整代码（逐段解释）

void rcc_sysclk_init(void)
{
    // ═══ 前置：Flash 等待周期 ═══
    FLASH->ACR |= (0x2 << 0);
    // 72MHz 太快，Flash 存储器跟不上
    // 需要插入 2 个等待周期，否则 CPU 读到错指令，直接卡死
    // ← 今天踩的坑：没这行 = 灯不闪！
    // ═══ 第①步：开 HSE 外部晶振，等它稳定 ═══
    RCC->CR |= (1 << 16);              // HSEON = 1 → 打开 8MHz 外部晶振
    while (!(RCC->CR & (1 << 17)));    // HSERDY = 1？没就绪就死等
    // 晶振起振需要时间（毫秒级），必须等
    // ═══ 第②步：配 PLL ═══
    RCC->CFGR |= (1 << 16);            // PLLSRC = 1 → PLL 时钟源选 HSE
    RCC->CFGR &= ~(1 << 17);           // PLLXTPRE = 0 → HSE 进 PLL 前不分频
    RCC->CFGR |= (0x7 << 18);          // PLLMUL = 0111 → ×9
    //    8MHz × 9 = 72MHz
    RCC->CR |= (1 << 24);              // PLLON = 1 → 开 PLL
    while (!(RCC->CR & (1 << 25)));    // PLLRDY = 1？没就绪就等
    // ═══ 第③步：配总线分频 ═══
    RCC->CFGR |= (0x4 << 8);           // PPRE1[10:8] = 100 → APB1 ÷2 = 36MHz
    // AHB 和 APB2 默认 ÷1，不用动
    // 为什么 APB1 要 ÷2？F103 硬件限制 APB1 不能超过 36MHz
    // ═══ 第④步：切换系统时钟到 PLL ═══
    RCC->CFGR |= (0x2 << 0);           // SW[1:0] = 10 → 选 PLL 为系统时钟
    while ((RCC->CFGR & 0xC) != 0x8);  // SWS[3:2] 变成 10 了？
    // 0xC = 1100 → 只保留 bit3-2
    // 0x8 = 1000 → SWS=10 = PLL
    // 不等于 → 还在切换中，等。等于 → 切换完成，走
}

调用方式

int main(void)
{
    rcc_sysclk_init();    // ← 第一行！先提频，后面所有操作都在 72MHz 下
    gpio_init(GPIOC, 13, OUT_PP);
    // ...
}

D8：SysTick 定时器 —— 从 "volatile 空循环" 到硬件精确延时

旧 delay vs 新 systick

旧 delay(volatile uint32_t n)新 systick_delay_ms(uint32_t ms)
精度不知道，全靠试精确到毫秒
换芯片重新瞎调 n改 LOAD 一个数
CPU空转，浪费电硬件计数，以后能用中断
依赖无需要 rcc_sysclk_init() 先配好 72MHz

SysTick 是什么？

SysTick 是 Cortex-M3 内核自带的 24 位硬件定时器——ARM 公司设计的，所有用 CM3 的芯片都有，不限于 STM32。它是一个向下计数器。

四个寄存器

寄存器干什么类比
SysTick->LOAD重装载值：从几开始往下数闹钟，设多长时间响
SysTick->VAL当前计数值：现在数到哪了闹钟还剩多少秒
SysTick->CTRL控制：开关、时钟源、标志位闹钟的开关+模式按钮
SysTick->CALIB校准值，不管它—

CTRL 寄存器关键位

位名称01
bit0ENABLE计数器停计数器跑
bit1TICKINT不触发中断数到 0 触发中断
bit2CLKSOURCEAHB/8（我们用的）AHB 直通
bit16COUNTFLAG还没数完数到 0 了！

核心问题：1ms 要数多少个时钟周期？

系统时钟 AHB = 72MHz
SysTick 用 AHB/8 = 9MHz
        = 每秒 9,000,000 个时钟周期
        ↓ ÷ 1000
1ms = 9,000 个周期

所以 LOAD = 9000
LOAD=9000 只在 72MHz 下成立。
如果你没调 rcc_sysclk_init()，芯片跑 8MHz：
AHB/8 = 1MHz → 1ms = 1000 周期 → LOAD 应该是 1000
→ 写 9000 就慢了 9 倍！← 下午刚踩的坑

完整代码（逐行解释）

void systick_delay_ms(uint32_t ms)
{
    // ① 选时钟源：bit2 = 0 → AHB/8 = 9MHz
    SysTick->CTRL &= ~(1 << 2);

    // ② 设 LOAD：1ms 要数 9000 个周期
    SysTick->LOAD = 9000;

    // ③ 启动计数器：bit0 = 1
    SysTick->CTRL |= 1;

    // ④ 循环 ms 次，每次等 1ms
    while (ms--) {
        // 内层：死等 COUNTFLAG 变 1
        // bit16 = 0（还没到）→ !0 = 真 → 继续等
        // bit16 = 1（到了！）→ !非零 = 假 → 往下走
        while (!(SysTick->CTRL & (1 << 16)));
        // 读到 bit16=1 的瞬间，硬件自动清零，VAL 自动重载 9000
    }

    // ⑤ 关计数器：bit0 = 0
    SysTick->CTRL &= ~1;
}

while (ms--) 怎么工作的？

调用 systick_delay_ms(500)：ms = 500

ms-- = 先用再减：
  第 1 轮：ms=500 ≠0 ✓ → ms 减到 499 → 等 1ms → 回头
  第 2 轮：ms=499 ≠0 ✓ → ms 减到 498 → 等 1ms → 回头
  ...
  第 500 轮：ms=1 ≠0 ✓ → ms 减到 0 → 等 1ms → 回头
  第 501 轮：ms=0 =0 ✗ → 退出

总共 = 500 × 1ms = 500ms ✅

完整链路：systick_delay_ms(500) 到底发生了什么

main.c 调用 systick_delay_ms(500) ↓ ms = 500 ↓ LOAD = 9000（硬件：每 9000 个周期 = 1ms） 启动计数器 ↓ 等 1ms → ms-1 → 等 1ms → ms-1 → ...（重复 500 次） ↓ 关计数器 → 回到 main.c

使用方式

#include "systick.h"            // ① 声明

systick_delay_ms(500);          // ② 500 毫秒
systick_delay_ms(10);           // ③ 10 毫秒
// 跟 gpio_write 一样：声明一次，后面直接调

🛠️ 调试日志：systick 踩坑实录

坑 1：LOAD = 9000 但芯片跑 8MHz

现象：灯闪得非常慢，500ms 延时感觉像 4-5 秒
原因：SystemInit 是空的，芯片跑 8MHz。LOAD=9000 是按 72MHz 算的
修复：写 rcc_sysclk_init()，把时钟拉到 72MHz→ LOAD=9000 自然就对了

坑 2：忘了 FLASH->ACR

现象：加了 rcc_sysclk_init() 之后灯完全不闪
原因：72MHz 下 Flash 需要 2 个等待周期，没设 → CPU 读指令出错 → 卡死
修复：FLASH->ACR |= (0x2 << 0); 加在 RCC 操作之前

坑 3：while 条件写反

错误写法：while (RCC->CR & (1 << 17)) → bit17=1 说明 HSE 就绪了，你却继续循环
正确写法：while (!(RCC->CR & (1 << 17))); → bit17=0（没就绪）→ 等。bit17=1 → 走
