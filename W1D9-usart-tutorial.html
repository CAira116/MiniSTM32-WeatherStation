0. 先回答你今晚最大的疑惑

❓ "我一直在这改代码，不知道到底干了什么"

答：你今晚写的 usart.c 就干了一件事——让 STM32 能对电脑说话。

三个函数各管各的：
· usart_init = 配好串口，准备说话（开机拨号）
· usart_send_byte = 发一个字（说一个字母）
· usart_send_string = 把一整句话一个字一个字发完

真正让你困惑的不是 USART，是 Keil 的 MicroLIB 配置和 printf 重定向——那是工程环境问题，不是你的代码问题。后面详细讲。

1. USART 到底是什么

1.1 用两根线聊天

STM32 PA9 (TX) ──────────→ RX USB-TTL 模块 ──→ 电脑串口助手 PA10 (RX) ←────────── TX

TX = 发送线（Transmit），RX = 接收线（Receive）。交叉连接：我的 TX 接你的 RX。

1.2 波特率 = 说话速度

115200 = 每秒发 115200 个 bit。一个字节约 10 个 bit（起始位+8数据位+停止位），所以每秒约发 11520 个字符。

1.3 数据格式：8N1

[起始位0] [bit0] [bit1] [bit2] [bit3] [bit4] [bit5] [bit6] [bit7] [停止位1] 1bit └────────── 8 位数据 ────────────────┘ 1bit = 8 个数据位 + 无校验 (N) + 1 个停止位

2. usart_init 逐行分解

void usart_init(USART_TypeDef *usart, uint32_t baud)
{
    // ═══ ① 开时钟：GPIOA + USART1 都要给电 ═══
    RCC->APB2ENR |= 1 << 2;        // bit2 = 1 → GPIOA 通电
    RCC->APB2ENR |= 1 << 14;       // bit14 = 1 → USART1 通电
为什么两个都要开？
USART 数据要从 PA9 引脚出去 → PA9 需要 GPIOA 能工作 → GPIOA 需要给电。
同时 USART 模块本身也需要自己的电。两把闸刀，一把不能少。
    // ═══ ② 配 GPIO 引脚 ═══
    gpio_init(GPIOA, 9, AF_PP);        // PA9 = TX = 复用推挽输出
    gpio_init(GPIOA, 10, IN_FLOAT);    // PA10 = RX = 浮空输入
模式引脚为什么
AF_PP 复用推挽PA9 (TX)引脚不归 GPIO 管了，交给 USART 控制。USART 通过它往外发数据。
IN_FLOAT 浮空输入PA10 (RX)只读外部电平，不做上拉下拉。电脑发来的信号读到就行。
    // ═══ ③ 配波特率 ═══
    usart->BRR = 72000000 / baud;       // 公式：APB2时钟 ÷ 波特率

BRR 是波特率寄存器（Baud Rate Register）。你对串口助手选了 115200，硬件就得按 115200 发——这个计算就是告诉硬件："数到 625 个时钟周期就切换一个 bit"。

波特率BRR 值公式
9600750072000000 / 9600
11520062572000000 / 115200
46080015672000000 / 460800
    // ═══ ④⑤ 使能 ═══
    usart->CR1 |= (1 << 3);        // TE = Transmit Enable → 发送功能开
    usart->CR1 |= (1 << 2);        // RE = Receive Enable  → 接收功能开
    usart->CR1 |= (1 << 13);       // UE = USART Enable    → 整个模块开
}
位名称01
CR1 bit2RE不收能接收数据
CR1 bit3TE不发能发送数据
CR1 bit13UE串口关串口开（总开关）

3. usart_send_byte 逐行分解 — 发一个字

void usart_send_byte(USART_TypeDef *usart, uint8_t data)
{
    while (!(usart->SR & (1 << 7)));   // 死等：DR 空了没？
    usart->DR = data;                    // 把数据塞进去，硬件自动发出
}

3.1 为什么 while 死等？

DR（数据寄存器）同一时刻只能放一个字节。
上一个 'H' 还在按 bit 往外搬（速度 115200 = 每个 bit 约 8.7 微秒，一个字节约 87 微秒），这时候往里塞 'e' → 两个字节混在一起 → 电脑端收到乱码。

3.2 SR bit7 = TXE（发送数据寄存器空）

TXE 状态含义操作
0DR 里还有数据，正在往外搬死等，不往里塞
1DR 空了可以写下一个字节了
while (!(usart->SR & (1 << 7)));

// 拆开读：
// (usart->SR & (1 << 7))  → TXE = 1？  → 1（真）或 0（假）
// !(真) = 假  → 退出循环（DR 空了，可以发了）  ← 走这条
// !(假) = 真  → 继续循环（DR 还忙着，等等）

3.3 TXE vs TC 的区别（今晚踩的坑）

标志位含义什么时候用
TXEbit7数据寄存器 DR 空了连续发多个字节时用
TCbit6发送彻底完成（DR空 + 移位寄存器也空）需要确认数据已经发到线上时用

DR 空了 ≠ 数据发完了。DR 空只表示"你可以塞下一个字节了"——但上一个字节可能还在移位寄存器里往外搬。TC=1 才表示"线路上都发完了"。

4. usart_send_string 逐行分解 — 发一整句

void usart_send_string(USART_TypeDef *usart, const char *str)
{
    while (*str) {                          // "指针指着的字符不是 '\0' 吗？"
        usart_send_byte(usart, *str);       // → 发这个字符
        str++;                              // → 指针往后移一格
    }
}

4.1 C 语言字符串是怎么存的

"Hi" 在内存里 = ['H', 'i', '\0'] [0] [1] [2] '\0' = 值 0 = 字符串结束标志（编译器自动加的）

4.2 while (*str) 的执行过程

调用 usart_send_string(USART1, "Hi");

str 指向 [0] = 'H'

第 1 轮：*str = 'H' ≠ 0  →  发 'H'  →  str 移到 [1]
第 2 轮：*str = 'i' ≠ 0  →  发 'i'  →  str 移到 [2]
第 3 轮：*str = '\0' = 0 → while(0) → 不进 → 退出 ✅

4.3 str++ 在干什么

str 是一个指针——存的是"当前字符在内存里的地址"。str++ 把地址加 1，指向下一个字符。

5. USB-TTL 接线（记得交叉！）

USB-TTL 模块方向STM32
RX←PA9 (TX)
TX→PA10 (RX)
GND—GND
⚠️ 常见错误：TX 接 TX、RX 接 RX。这样永远通不了——你说话接对方耳朵，不是接对方嘴巴。交叉连接是串口的物理铁律。

6. 今晚踩的坑：printf 重定向 + MicroLIB

6.1 我想做什么

让 printf("Hello %d\r\n", 42); 自动从串口输出——不用手动调 usart_send_string。

6.2 需要什么

写一个 fputc 函数，编译器遇到 printf 时自动调它来发每个字符。然后用 Keil 的 MicroLIB（精简 C 库）。

6.3 为什么卡住了

MicroLIB 和当前工程的启动文件（startup_stm32f103xb.s）不完全兼容。
启动文件引用了 __use_two_region_memory 和 __initial_sp 两个符号，MicroLIB 不提供它们 → 链接失败。

6.4 解决方案（明天做）

  1. 换一个兼容 MicroLIB 的启动文件，或
  2. 不用 printf，直接用 usart_send_string + sprintf 组合，或
  3. 配好 scatter file（分散加载文件）

这不是你的代码问题。是工程配置层面的兼容性问题。

7. USART 寄存器速查表

寄存器全称干什么今天用的位
USART->BRR波特率寄存器72MHz / 你要的波特率全部 16 位
USART->SR状态寄存器看发完了没 / 收到了没bit7=TXE（DR空）、bit6=TC（全发完）、bit5=RXNE（收到数据）
USART->DR数据寄存器往里写=发送 / 从里读=接收低 9 位（8bit 数据+1bit 校验）
USART->CR1控制寄存器1开关和模式bit13=UE（总开关）、bit3=TE（发送）、bit2=RE（接收）

8. 疑问解答汇总

❓ RCC 到底管什么？
RCC = 芯片总配电房。管两件事：
① 系统跑多快（rcc_sysclk_init）
② 每个外设给不给电（rcc_enable）
忘开时钟 = 外设在睡觉 = 代码全对也不工作。
❓ 为什么寄存器位号记不住？
正常！嵌入式工程师不背寄存器位——翻手册。翻多了手指会自己找位置。每次写代码把位号写在注释里（你 gpio.c 和 usart.c 已经有这个习惯了）。
❓ usart_init 里的时钟为什么要开两个？
GPIOA → PA9/PA10 引脚能工作
USART1 → 串口模块本身能工作
两个独立的外设，两把独立电闸，都要推。
❓ while (!(SR & (1<<7))) 到底在等什么？
等数据寄存器 DR 变空。DR 只能放一个字节，上一个没发完不能塞新的。等 TXE=1 表示"仓库空了，可以进货了"。
❓ gpio_write 能写 USART 寄存器吗？
不能。gpio_write 是往 GPIO 引脚写高低电平的。USART 寄存器直接用位操作（|=、&= ~）。不同外设、不同操作方式。
❓ 从空白 main.c 写代码为什么这么难？
你今天上午到晚上从"不会写"到独立写了 SOS + 心跳灯 + 摩尔电码三个完整程序——就是在练这个。难是因为每次都要自己做所有决定，但这就是"会写代码"和"会改代码"的区别。继续练。
