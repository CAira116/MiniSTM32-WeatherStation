## 速查总表

### 常用寄存器地址

| 外设 | 基地址 | 常用寄存器偏移 |
|------|--------|---------------|
| RCC | `0x40021000` | APB2ENR=`+0x18`, APB1ENR=`+0x1C` |
| GPIOA | `0x40010800` | CRL=`+0x00`, CRH=`+0x04`, IDR=`+0x08`, ODR=`+0x0C` |
| GPIOB | `0x40010C00` | 同上 |
| AFIO | `0x40010000` | MAPR=`+0x04`, EXTICR[0~3]=`+0x08~0x14` |
| TIM2 | `0x40000000` | CR1=`+0x00`, SR=`+0x10`, DIER=`+0x0C`, PSC=`+0x28`, ARR=`+0x2C` |
| USART1 | `0x40013800` | SR=`+0x00`, DR=`+0x04`, BRR=`+0x08`, CR1=`+0x0C` |
| NVIC | `0xE000E100` | ISER[0]=`+0x00`, ISER[1]=`+0x04` |

### 位操作速查

| 操作 | 写法 | 用途 |
|------|------|------|
| 某位置 1 | `reg \|= (1 << n)` | 开时钟、开中断、拉高引脚 |
| 某位清零 | `reg &= ~(1 << n)` | 关外设、清标志、拉低引脚 |
| 某位翻转 | `reg ^= (1 << n)` | LED 闪烁 |
| 读某位 | `(reg >> n) & 1` 或 `reg & (1 << n)` | 判断按键、查标志 |
| 修改多位 | `(reg & ~mask) \| value` | 配置引脚模式、定时器参数 |

## 一、GPIO 寄存器

### 每个 GPIO 端口有 7 个寄存器

| 寄存器 | 全称 | 作用 |
|--------|------|------|
| CRL | 端口配置低寄存器 | 配置 Px0~Px7（每引脚 4 位） |
| CRH | 端口配置高寄存器 | 配置 Px8~Px15（每引脚 4 位） |
| IDR | 输入数据寄存器 | 读引脚电平（输入用） |
| ODR | 输出数据寄存器 | 写引脚电平（输出用） |
| BSRR | 位设置/清除寄存器 | 原子操作置位/清零 |

### 每个引脚 4 位配置

```
每个引脚的 4 位 = CNF[1:0] + MODE[1:0]

MODE ≠ 00（输出模式）：
  CNF=00 → 通用推挽输出
  CNF=01 → 通用开漏输出
  CNF=10 → 复用功能推挽输出
  CNF=11 → 复用功能开漏输出
  MODE=01 → 10MHz
  MODE=10 → 2MHz
  MODE=11 → 50MHz

MODE=00（输入模式）：
  CNF=00 → 模拟输入
  CNF=01 → 浮空输入
  CNF=10 → 上拉/下拉输入
```

### CNF/MODE 完整对照表

每脚 4 位 = `[CNF1 CNF0 MODE1 MODE0]`（高位→低位）

**输出模式（MODE ≠ 00）：**

| 速度 | MODE | CNF=00 推挽 | CNF=01 开漏 | CNF=10 复用推挽 | CNF=11 复用开漏 |
|------|------|------------|------------|----------------|----------------|
| 10MHz | 01 | `0x1` | `0x5` | `0x9` | `0xD` |
| 2MHz | 10 | `0x2` | `0x6` | `0xA` | `0xE` |
| **50MHz** | 11 | **`0x3`** | `0x7` | `0xB` | `0xF` |

**输入模式（MODE = 00）：**

| CNF | 4位值 | 含义 | 用途 |
|-----|--------|------|------|
| 00 | `0x0` | 模拟输入 | ADC |
| 01 | `0x4` | 浮空输入 | 按键检测（需要外部上拉） |
| 10 | `0x8` | 上拉/下拉输入 | 矩阵键盘列输入 |
| 11 | `0xC` | 保留 | — |

### 你最常用的几个

| 场景 | 值 | 拆解 |
|------|----|------|
| 数码管段码/位选 | `0x3` | CNF=00推挽 MODE=11 50MHz |
| 矩阵键盘行输出 | `0x3` | 同上 |
| 矩阵键盘列输入 | `0x8` | CNF=10上拉 MODE=00输入 |
| 按键/开关输入 | `0x4` | CNF=01浮空 MODE=00输入 |

### CRL 寄存器（Px0~Px7）

```
bit 31~28  bit 27~24  bit 23~20  bit 19~16
  Px7        Px6        Px5        Px4
bit 15~12  bit 11~8   bit 7~4    bit 3~0
  Px3        Px2        Px1        Px0
```

### CRH 寄存器（Px8~Px15）

```
bit 31~28  bit 27~24  bit 23~20  bit 19~16
  Px15       Px14       Px13       Px12
bit 15~12  bit 11~8   
  Px11       Px10
bit 7~4    bit 3~0
  Px9        Px8
```

### ODR：一次管 16 个引脚

```
GPIOA->ODR 是 16 位寄存器：
  bit15~bit8  → PA8~PA15（段选）
  bit7~bit0   → PA0~PA7（位选）

要动低 8 位 → 先 & 0xFF00 保留高 8 位
要动高 8 位 → 先 & 0x00FF 保留低 8 位
```

### 拉高/拉低

| 操作 | 含义 | 代码 |
|------|------|------|
| 拉高 | 引脚输出 3.3V | `ODR \|= (1 << n)` |
| 拉低 | 引脚输出 0V(GND) | `ODR &= ~(1 << n)` |

## 二、RCC 时钟

### APB2ENR 寄存器

STM32 上电后外设默认断电省电，要先开时钟。

| 位 | 外设 | 功能 |
|----|------|------|
| bit 0 | AFIO | 复用功能 IO，关 JTAG 必开 |
| bit 2 | GPIOA | PA 口时钟 |
| bit 3 | GPIOB | PB 口时钟 |
| bit 4 | GPIOC | PC 口时钟 |
| bit 5 | GPIOD | PD 口时钟 |
| bit 9 | ADC1 | 模数转换器 1 |
| bit 12 | SPI1 | 串行外设接口 1 |
| bit 14 | USART1 | 串口 1 |

### APB1ENR 寄存器

| 位 | 外设 | 功能 |
|----|------|------|
| bit 0 | TIM2 | 通用定时器 2 |
| bit 1 | TIM3 | 通用定时器 3 |
| bit 2 | TIM4 | 通用定时器 4 |
| bit 17 | USART2 | 串口 2 |
| bit 21 | I2C1 | I2C 接口 1 |

### 开时钟的标准写法

```c
RCC->APB2ENR |= 1<<0 | 1<<2;    // AFIO + GPIOA（数码管）
RCC->APB2ENR |= 1<<3;           // GPIOB（键盘/LED）
RCC->APB1ENR |= 1<<0;           // TIM2（定时器）
RCC->APB2ENR |= 1<<14;          // USART1（串口）
// 用 |= 不用 = 的原因：不能盖掉其他位（可能有别的外设已经开了钟）
```

## 三、矩阵键盘

### 硬件连接

```
PB0~PB3 → 行线（输出）
PB4~PB7 → 列线（输入）
```

### 行输出、列输入的原因

- 行线主动拉低，制造检测信号（一次只拉低一行）
- 列线被动检测，看有没有被拉低
- 按键按下 → 行线和列线导通 → 拉低的电压传到列线 → 读到 0

### 行扫描的四个值

```
0x000E = 1110 → PB0=0（行0拉低）
0x000D = 1101 → PB1=0（行1拉低）
0x000B = 1011 → PB2=0（行2拉低）
0x0007 = 0111 → PB3=0（行3拉低）
```

### 上拉电阻的作用

列线 PB4~PB7 设为上拉输入（CNF=10 MODE=00）：

```
没按下：上拉电阻把列线维持在 3.3V → IDR 读到 1
按下：  行线的 0V 通过按键把列线拽到 0V → IDR 读到 0
```

### IDR：读取引脚实际电平

```c
GPIOB->IDR   // 16 位寄存器，每一位 = 对应引脚上的真实电压
             // bit4=PB4, bit5=PB5, ... bit15=PB15
             // 1 = 3.3V, 0 = 0V
```

### 读单个引脚：IDR & (1<<n)

```c
(GPIOB->IDR & (1<<4)) == 0   // 只看 bit4（PB4）
// & (1<<4) = 把其他位全挡住，只看第 4 位
// == 0     = 引脚是低电平 0V → 被拉低了 → 有按键
```

### 按键编码规则

| 按键位置 | 返回值 | 含义 |
|---------|--------|------|
| 行0 列0 | 0x10 | 十位=行号, 个位=列号 |
| 行0 列1 | 0x11 | |
| 行1 列0 | 0x14 | |
| 行3 列3 | 0x1F | 最后一个键 |

### 扫描过程

```
① 行0 拉低，行1~3 拉高 → Delay(200) → 读列线 → 4 次 if 检测
② 行1 拉低，行0,2,3 拉高 → Delay(200) → 读列线 → 4 次 if 检测
③ 行2 拉低 → ...
④ 行3 拉低 → ...
```

## 四、数码管显示

### 硬件连接

```
PA0~PA7  → 段选（控制 a b c d e f g dp 哪个笔画亮）
PA8~PA15 → 位选（控制 8 个数码管中哪一个亮）
```

### 8 段数码管

```
      a
    ┌───┐
   f│ g │b
    ├───┤
   e│   │c
    └───┘
      d     ●dp
```

### 段码表（共阴：1 亮 0 灭）

| 数字 | a b c d e f g | 十六进制 |
|------|--------------|----------|
| 0 | 1 1 1 1 1 1 0 | 0x3F |
| 1 | 0 1 1 0 0 0 0 | 0x06 |
| 2 | 1 1 0 1 1 0 1 | 0x5B |
| 3 | 1 1 1 1 0 0 1 | 0x4F |
| 4 | 0 1 1 0 0 1 1 | 0x66 |
| 5 | 1 0 1 1 0 1 1 | 0x6D |
| 6 | 1 0 1 1 1 1 1 | 0x7D |
| 7 | 1 1 1 0 0 0 0 | 0x07 |
| 8 | 1 1 1 1 1 1 1 | 0x7F |
| 9 | 1 1 1 1 0 1 1 | 0x6F |

### 位选：每次只亮一个数码管

```c
LedBit[8] = {0x80, 0x40, 0x20, 0x10, 0x08, 0x04, 0x02, 0x01};
//           第1位  第2位  第3位  第4位  第5位  第6位  第7位  第8位
```

### 显示一个数字的完整流程

```c
// 1. 先写位选（选中第 i 个数码管），同时保留段选数据
GPIOA->ODR = (GPIOA->ODR & 0xFF00) | LedBit[i];

// 2. 再写段选（显示要显示的笔画），同时保留位选数据
GPIOA->ODR = (GPIOA->ODR & 0x00FF) | LedBuf[i] << 8;

// 3. 延时 500 微秒 → 换下一位 → 循环
//    人眼视觉暂留 → 看起来 8 个数码管同时亮
```

## 五、易错点强化（复习题复盘 2026-06-05）

### 1. 引脚配置四位的含义——CNF vs MODE（Q16/Q18 强化）

每个 GPIO 引脚用 **4 位**配置，这 4 位 = **CNF[1:0] + MODE[1:0]**。

```
一个引脚的 4 位配置：
bit[3:2] = CNF（配置模式）
bit[1:0] = MODE（速度/输入输出）

MODE ≠ 00（输出模式时）：
  CNF=00 → 通用推挽输出（最常用）
  CNF=01 → 通用开漏输出
  CNF=10 → 复用功能推挽输出
  CNF=11 → 复用功能开漏输出
  MODE=01 → 10MHz
  MODE=10 → 2MHz
  MODE=11 → 50MHz

MODE=00（输入模式时）：
  CNF=00 → 模拟输入
  CNF=01 → 浮空输入（电平不确定，需要外部电路决定）
  CNF=10 → 上拉/下拉输入（内部电阻拉高或拉低）
```

两个最常用的配置值：

| 值 | 二进制 CNF,MODE | 拆解 | 含义 |
|----|----------------|------|------|
| `0x3` | 00,11 | CNF=00（推挽）MODE=11（50MHz） | **推挽输出 50MHz** |
| `0x8` | 10,00 | CNF=10（上拉/下拉）MODE=00（输入） | **上拉/下拉输入** |

> 🔑 **记法**：MODE 先看是不是 00，是=输入，不是=输出。然后 CNF 决定具体模式。

### 2. 为什么必须先开 RCC 时钟？（Q19 强化）

STM32 为了省电，上电后**所有外设时钟默认关闭**。不开时钟 = 外设内部电路不工作 = 寄存器写不进去。

```
类比：RCC 时钟 = 给外设供电
不开钟 → GPIO 没通电 → 配 CRL、写 ODR 全是徒劳

RCC->APB2ENR |= 1<<2;   // 先给 GPIOA 通上电
GPIOA->CRL = ...;        // 然后才能配置
```

> 🔴 忘了开 RCC = 最常见的坑。代码写对了但 GPIO 不工作 → 第一反应检查 RCC 开了没。

### 3. 矩阵键盘——行输出、列输入的原因（Q20 强化）

```
硬件连接：PB0~PB3 = 行线（输出），PB4~PB7 = 列线（输入）

       PB4    PB5    PB6    PB7  （列 = 输入）
        │      │      │      │
PB0 ───┼──────┼──────┼──────┼───  （行0 = 输出）
PB1 ───┼──────┼──────┼──────┼───  （行1 = 输出）
PB2 ───┼──────┼──────┼──────┼───  （行2 = 输出）
PB3 ───┼──────┼──────┼──────┼───  （行3 = 输出）

按键就是行线和列线的交叉点。
按下 → 行线和列线短路导通。
```

- **行 = 输出**：主动拉低，只有一行被拉低（制造检测信号）
- **列 = 输入**：被动检测——平时被上拉电阻维持在 3.3V（读=1），按下时被行线的 0V 拽低（读=0）

> 一主动一被动。如果反过来也行，但这是标准做法。

### 4. 上拉电阻——没有它，键盘废了（Q21 强化）

```
不加（浮空）：
  没按时 → 列线电压不定（受电磁干扰）→ IDR 读到 0 或 1 → 乱跳
  按  下 → 列线 = 0V → 读到 0

加上拉：
  没按时 → 上拉电阻把列线拉到 3.3V → IDR 稳定读到 1
  按  下 → 行线 0V 把列线硬拽到 0V → IDR 读到 0
```

**上拉电阻的作用**：给输入引脚一个确定的默认电平（1），只有按键按下时才变 0。代码里 `CNF=10 MODE=00`（上拉/下拉输入）就是干这个的。

> 对比：上拉 = 默认高；下拉 = 默认低。键盘用上拉是因为行输出低电平方便检测。

### 5. 按键消抖——pre_key 状态机（Q23 强化）

机械按键按下和松开时，触点会弹跳（几毫秒内反复通断），不处理的话一次按键被当成多次。

```c
// 消抖三步：
pre_key = KeyScan();           // ① 第一次读
if (pre_key == 0 && key != 0)  // ② 判断：上次没键→这次有键
{
    Delay(5500);               // ③ 延时等抖完
    key = KeyScan();           // ④ 再读一次确认
    if (key == pre_key)        // ⑤ 两次一致才算数
        // 处理按键
}
```

**pre_key 的作用**：
- 存**上一轮**的键值
- 只有 `pre_key == 0（无键） && 当前有键` 才算"新按下"
- 配合延时 + 二次读取 → 过滤掉弹跳的抖动

| 场景 | pre_key | 当前 key | 判定 |
|------|---------|----------|------|
| 空闲 | 0 | 0 | 无事 |
| 按下瞬间 | 0 | 非0 | ✅ 新按键！ |
| 按住不放 | 非0 | 非0 | 忽略（不是新按） |
| 抖动 | 0 | 乱变 | Delay 后二次读 → 稳定了才算 |

### 6. 数码管——段选 vs 位选，人眼骗局（Q24 强化）

```
PA0~PA7   = 段选 → 控制 a~g + dp 哪段亮
PA8~PA15  = 位选 → 控制 8 个数码管哪个亮

      段选线（8根，所有数码管共用）
      ═══════════════════════
      ║  a b c d e f g dp   ║
      ║  ┌──────────────┐   ║
      ║  │  数码管 0    │   ║ ← 位选0，只开这一个
      ║  ├──────────────┤   ║
      ║  │  数码管 1    │   ║
      ║  ├──────────────┤   ║
      ║  │  ...         │   ║
      ║  └──────────────┘   ║
      ═══════════════════════
```

**为什么交替切换**：8 个数码管共用同一套段选线（PA0~PA7），同一时刻所有数码管看到的是同一组段选数据。如果 8 个同时亮 → 每个都显示同一个数字 → 没法用。

**人眼骗局**：
1. 位选只开第 0 个 → 段选写第 0 个数字 → 延时 500μs
2. 位选只开第 1 个 → 段选写第 1 个数字 → 延时 500μs
3. ...循环 8 次 → 回到第 1 步

```
切换频率 ≈ 1 / (8 × 500μs) = 250Hz ≫ 人眼临界频率 50Hz
→ 人眼看到的是 8 个数字"同时"亮
→ 实际上是轮流亮的，只是太快了看不出来
```

> 🎯 **一句话**：段选决定"显示什么"，位选决定"显示在哪个管"。轮流切换 + 视觉暂留 = 看起来全部同时亮。

## 六、TIM2 定时器

### TIM2 寄存器速查

| 寄存器 | 作用 | 关键位 |
|--------|------|--------|
| `TIM2->PSC` | 预分频器 | 16 位，实际分频 = PSC+1 |
| `TIM2->ARR` | 自动重装载 | 16 位，数到 ARR 就溢出 |
| `TIM2->CR1` | 控制寄存器 1 | bit0=CEN（使能计数器） |
| `TIM2->SR` | 状态寄存器 | bit0=UIF（更新中断标志） |
| `TIM2->DIER` | 中断使能寄存器 | bit0=UIE（更新中断使能） |
| `TIM2->EGR` | 事件生成寄存器 | bit0=UG（手动产生更新） |

### PSC / ARR 计算公式

```c
// 溢出周期 = (PSC + 1) × (ARR + 1) / 系统时钟

// 例1：1ms 溢出（方式一）
TIM2->PSC = 7200 - 1;  // 72MHz / 7200 = 10kHz → 每计数 0.1ms
TIM2->ARR = 10 - 1;    // 10 × 0.1ms = 1ms

// 例2：1ms 溢出（方式二）
TIM2->PSC = 720 - 1;   // 72MHz / 720 = 100kHz → 每计数 0.01ms
TIM2->ARR = 100 - 1;   // 100 × 0.01ms = 1ms
```

### 轮询方式（完整模板）

```c
void Tim2_Init(void) {
    RCC->APB1ENR |= 1<<0;     // TIM2 时钟
    TIM2->PSC = 7200 - 1;
    TIM2->ARR = 10 - 1;
    TIM2->EGR |= 1<<0;        // 手动更新，立即加载 PSC/ARR
    TIM2->CR1 |= 1<<0;        // 启动
}

int main(void) {
    Tim2_Init();
    while (1) {
        if ((TIM2->SR & (1<<0)) == (1<<0)) {  // 溢出标志位
            TIM2->SR &= ~(1<<0);              // 手动清标志
            // 做你想做的事
        }
    }
}
```

### 中断方式（完整模板）

```c
void Tim2_Init(void) {
    RCC->APB1ENR |= 1<<0;
    TIM2->PSC = 7200 - 1;
    TIM2->ARR = 10 - 1;
    TIM2->DIER |= 1<<0;                             // ① 开更新中断
    NVIC->ISER[TIM2_IRQn / 32] |= 1 << (TIM2_IRQn % 32);  // ② NVIC 使能
    TIM2->EGR |= 1<<0;
    TIM2->CR1 |= 1<<0;
}

void TIM2_IRQHandler(void) {            // ③ 函数名必须匹配启动文件
    static int i = 0;                   // static 保持值不丢失
    if ((TIM2->SR & (1<<0)) == (1<<0)) {
        // 做你想做的事
        TIM2->SR &= ~(1<<0);
    }
}

int main(void) {
    Tim2_Init();
    while (1) { ; }   // 空循环，中断干活
}
```

### NVIC->ISER 那行在算什么

```
TIM2_IRQn = 28（中断号）

28 / 32 = 0  → ISER[0]（哪个 32 位寄存器）
28 % 32 = 28 → bit 28（寄存器里第几位）

翻译：ISER[0] 的 bit28 = 1 → CPU 接收 28 号中断
```

### 轮询 vs 中断

| 项目 | 轮询 | 中断 |
|------|------|------|
| CPU 占用 | 100%（一直问） | 几乎 0（中断才干活） |
| 实时性 | 差（while 循环有延迟） | 好（立即响应） |
| 写起来 | 简单 | 多配两个寄存器 |
| 适用 | 只有一件事要做 | 多任务并行 |

## 七、数码管 + 键盘完整项目模板

### 硬件连接总览

```
PA0~PA7   → 数码管位选（8位）
PA8~PA15  → 数码管段码（a~g+dp）
PB0~PB3   → 矩阵键盘行（输出）
PB4~PB7   → 矩阵键盘列（输入，需上拉）
```

### 三个核心数组的关系

```c
unsigned char LedSeg[11] = {0x3F, ..., 0x6F, 0x40};  // 段码表（0~9 + 横杠）
unsigned char LedBit[8]  = {0x80, ..., 0x01};         // 位选表
unsigned char LedBuf[8]  = {10, ..., 10};             // 显示缓冲（动态变化！）

// 扫描时查表路径：
// LedBuf[i] → 数字（0~9, 10=横杠）
// LedSeg[LedBuf[i]] → 段码
// LedBit[i] → 位选
```

### LedBuf 的生命周期

```
开机       → 全 10 → 8 位横杠 "--------"
按数字键   → LedBuf[cnt] = 数字（0~9）→ 对应位变成数字
退格       → cnt--, LedBuf[cnt] = 10 → 恢复横杠
清零       → 全填 10, cnt = 0       → 恢复 "--------"
```

### GPIOC->CRL 混合配置：0x88883333

```
GPIOB->CRL = 0x88883333;

0x88883333 拆开：
  PB7: 1000 = 0x8 = 上拉/下拉输入（列3）
  PB6: 1000 = 0x8 = 上拉/下拉输入（列2）
  PB5: 1000 = 0x8 = 上拉/下拉输入（列1）
  PB4: 1000 = 0x8 = 上拉/下拉输入（列0）
  PB3: 0011 = 0x3 = 推挽输出 50MHz（行3）
  PB2: 0011 = 0x3 = 推挽输出 50MHz（行2）
  PB1: 0011 = 0x3 = 推挽输出 50MHz（行1）
  PB0: 0011 = 0x3 = 推挽输出 50MHz（行0）

别忘了开上拉：
  GPIOB->ODR |= 0x00F0;   // PB4~PB7 上拉
```

      pre_key=0     ┌── 进if，处理按键

## 八、USART 串口

### USART1 寄存器速查

| 寄存器 | 作用 | 关键位 |
|--------|------|--------|
| `USART1->BRR` | 波特率寄存器 | 波特率 = 时钟频率 / BRR（例：72M/9600=7500） |
| `USART1->CR1` | 控制寄存器 1 | bit13=UE, bit3=TE, bit2=RE |
| `USART1->SR` | 状态寄存器 | bit7=TXE（发送空）, bit6=TC（发送完成）, bit5=RXNE（收到数据） |
| `USART1->DR` | 数据寄存器 | 写=发送, 读=接收 |

### USART1 初始化模板（PA9 TX, PA10 RX, 9600bps）

```c
void Usart1_Init(void) {
    RCC->APB2ENR |= 1<<14;        // USART1 时钟
    RCC->APB2ENR |= 1<<2;         // GPIOA 时钟
    
    // PA9=TX 复用推挽输出, PA10=RX 浮空输入
    GPIOA->CRH = (GPIOA->CRH & 0xFFFFF00F) | 0x000004B0;
    //             PA9: 1011=0xB       PA10: 0100=0x4
    
    USART1->BRR = 72000000 / 9600; // 波特率 9600
    USART1->CR1 |= 1<<13 | 1<<3 | 1<<2;  // UE + TE + RE
}
```

### 发送一个字符

```c
void Usart1_SendChar(unsigned char ch) {
    USART1->DR = ch;                    // 写入数据
    while ((USART1->SR & (1<<6)) == 0); // 等 TC=1（发送完成）
    USART1->SR &= ~(1<<6);              // 手动清 TC
}
```

### 串口 PA9/PA10 引脚配置解析

```
PA9  = TX → 复用推挽输出 50MHz → CNF=10 MODE=11 → 1011 = 0xB
PA10 = RX → 浮空输入           → CNF=01 MODE=00 → 0100 = 0x4

0x000004B0 = 0000 0000 0000 0000 0000 0100 1011 0000
                                                 ││││
                                       PA10=0100 ─┘│││
                                        PA9=1011 ───┘││
                                       PA8=0000 ─────┘│
                                       PA11=0000 ──────┘
```

## 九、常见 C 语言坑（考试必考）

### 1. `=` vs `==` vs `|=`

```c
i = 0;     // 赋值
i == 0;    // 比较（用在 if 里）
reg |= 1;  // 或等于 → 只置位不动别的
reg = 1;   // 直接赋值 → 其他位全清零！
```

### 2. if 不加 {} 只控一条

```c
if (cnt < 8)
    LedBuf[cnt] = key;   // ← 这条归 if 管
    cnt++;               // ← 这条永远执行！if 管不到它

// 正确：
if (cnt < 8) {
    LedBuf[cnt] = key;
    cnt++;
}
```

### 3. `static` vs 局部变量

```c
void ISR(void) {
    int i = 0;         // 每次进中断都重新 = 0
    static int j = 0;  // 只初始化一次，值保留
}
```

### 4. 运算符优先级

```c
A & B == C   →  实际是 A & (B == C)  // & 优先级低于 ==
// 正确：
(A & B) == C
```
