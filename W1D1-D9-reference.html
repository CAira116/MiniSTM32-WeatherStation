D1-D3 · 环境搭建 & 寄存器版 Blinky

核心概念：GPIO 是谁？

STM32 芯片外面有一排金属引脚，GPIO（General Purpose Input/Output）是控制这些引脚的外设模块。你可以把任意引脚配置成输入（读外面电压）或输出（自己产生 3.3V/0V）。

四步点灯法（寄存器版）

// ① 开 GPIO 口时钟 —— 不给电，模块是死的
RCC->APB2ENR |= 1 << 2;           // GPIOA 挂在 APB2 总线 bit2 上

// ② 清空目标引脚配置位
GPIOA->CRL &= ~(0xF << 20);       // PA5 在 CRL 的 bit20-23，先清零

// ③ 写入模式
GPIOA->CRL |= (0x3 << 20);        // 0x3 = 0011 = 推挽输出 50MHz

// ④ 输出高低电平
GPIOA->BSRR = 1 << 5;             // PA5 高 → LED 亮
GPIOA->BSRR = (1 << (5+16));      // PA5 低 → LED 灭
🔑 嵌入式第一咒语：时钟 → 模式 → 数据，三步走。忘开时钟 = 代码全对但什么都不发生，调三天找不出原因。

五大 GPIO 寄存器速查

寄存器全称干啥用比喻
CRL端口配置低寄存器pin 0-7 的模式（输入/输出/速度）遥控器上排按钮
CRH端口配置高寄存器pin 8-15 的模式遥控器下排按钮
BSRR端口位设置/清除寄存器拉高/拉低引脚（原子操作）瞬时开关，写1就生效
IDR端口输入数据寄存器只读，读取引脚当前电平电压表
ODR端口输出数据寄存器读/写，当前输出电平输出状态记录本
⚠️ volatile 的坑：指向寄存器的指针必须用 volatile。因为寄存器的值不受 CPU 控制（硬件自己会改），不加 volatile 编译器可能把连续两次读取优化成一次，读不到真实变化。
#define GPIOA ((GPIO_TypeDef *)0x40010800) 里的 GPIO_TypeDef 所有成员都是 volatile 的——STM32 头文件已经帮你加好了。

D4 · 按键输入（轮询）

输入模式 vs 输出模式

模式电流流向什么时候用
浮空输入高阻态，引脚悬空时电平不确定外部电路已确定电平
上拉输入内部电阻拉到 VDD（3.3V），默认高按键一脚接引脚，一脚接 GND
下拉输入内部电阻拉到 GND，默认低按键一脚接引脚，一脚接 VDD

按键电路（面包板）

引脚 PA0 ──┬── 按键 ── GND
           │
        内部上拉电阻（~40kΩ）→ VDD 3.3V

没按：PA0 被上拉电阻拉到 3.3V → IDR bit0 = 1
按下：PA0 直接通 GND → IDR bit0 = 0

轮询代码

// 初始化 PA0 为上拉输入
RCC->APB2ENR |= 1 << 2;
GPIOA->CRL |= (0x8 << 0);       // CNF=10 MODE=00 = 0x8 = 上拉输入

// 循环读按键（轮询）
while (1) {
    if ((GPIOA->IDR & (1 << 0)) == 0) {   // 读到 0 = 按键按下
        // 做点什么
    }
}

ODR 的双重身份（输入模式下）

同一个 ODR 寄存器，输出模式下决定输出电平，输入模式下开关内部上拉/下拉电阻：

GPIOA->ODR |= (1 << 0);   // 输入模式下 → ODR bit0=1 → 启用上拉
GPIOA->ODR &= ~(1 << 0);  // 输入模式下 → ODR bit0=0 → 启用下拉

D5 · 外部中断 EXTI

为什么不用轮询？

轮询 = CPU 不停地在问"按键按了没？按键按了没？"——浪费电，占着 CPU 不让干别的事。
中断 = CPU 干自己的事，按键按下时硬件自动打断 CPU，处理完再回去。

EXTI 六步配置流程（背下来）

// ① 开 GPIO 时钟
RCC->APB2ENR |= 1 << 2;               // GPIOA

// ② 开 AFIO 时钟（中断要用的"接线板"）
RCC->APB2ENR |= 1 << 0;               // AFIO

// ③ 配置 GPIO 为输入
GPIOA->CRL |= (0x8 << 0);             // PA0 上拉输入

// ④ AFIO 接线：把 PA0 连到 EXTI0 线
AFIO->EXTICR[0] &= ~(0xF << 0);       // EXTI0 ← PA0

// ⑤ 配置 EXTI：上升沿/下降沿/双边沿触发
EXTI->IMR  |= (1 << 0);               // 不屏蔽 EXTI0
EXTI->RTSR |= (1 << 0);               // 上升沿触发（松开按键时触发）

// ⑥ NVIC 使能：允许这个中断打断 CPU
NVIC->ISER[0] = (1 << 6);             // EXTI0 的中断号是 6（要查手册！）
NVIC->IP[6] = (0x01 << 4);            // 优先级设为 1

EXTI 框架：三层漏斗

AFIO（接线板） → EXTI（检测器） → NVIC（总闸） 把指定引脚 检测边沿（上升/ 允许/禁止中断 连到 EXTI 线 下降/双边沿） 送到 CPU

中断服务函数

// 函数名必须和启动文件里的弱定义完全一致！写错一个字母=不触发
void EXTI0_IRQHandler(void)
{
    if (EXTI->PR & (1 << 0)) {          // 检查 EXTI0 是否触发了
        EXTI->PR |= (1 << 0);           // 写 1 清中断标志（必须清！）
        // 你的处理代码
    }
}
⚠️ 中断函数一定要清标志位！
EXTI->PR 写 1 清零（和 BSRR 一样的设计）。不清 = 中断函数不断重入，MCU 卡死。

D6 · MiniLib gpio.c 封装（今天的内容）

详见完整教程：W1D6-gpio-tutorial.html  |  自测：W1D6-self-test.html

四个函数一句话总结

函数一句话核心寄存器
gpio_init设置引脚是输入还是输出、什么模式RCC_APB2ENR + CRL/CRH
gpio_write指定引脚输出高或低BSRR
gpio_read看引脚现在高还是低，返回 0/1IDR
gpio_toggle高变低、低变高，来回翻ODR → BSRR

gpio_init 核心逻辑：CRL vs CRH 的偏移计算

CRL (pin 0-7) ：每个 pin 占 4 位 → offset = pin × 4 CRH (pin 8-15)：每个 pin 占 4 位 → offset = (pin - 8) × 4 pin5 → CRL bit20-23 (5 × 4 = 20) pin10 → CRH bit8-11 ((10-8) × 4 = 8)

BSRR 原子操作原理

低 16 位写 1 → 引脚拉高     例：BSRR = (1 << 5)  → PA5 High
高 16 位写 1 → 引脚拉低     例：BSRR = (1 << 21) → PA5 Low  (21 = 5+16)
写 0 到任何位 → 什么都不发生

D7 · rcc_sysclk_init — 系统时钟 8MHz → 72MHz

详见教程：W1D7-D8-tutorial.html

时钟路径

HSE 8MHz → PLL ×9 → 72MHz → SYSCLK ├→ AHB ÷1 = 72MHz ├→ APB2 ÷1 = 72MHz └→ APB1 ÷2 = 36MHz（最大36）

四步配时钟（背下来）

// 前置：Flash 等待周期！忘了 = 芯片卡死
FLASH->ACR |= (0x2 << 0);            // 72MHz 需要 2 个等待周期

// ① 开 HSE，等稳定
RCC->CR |= (1 << 16);
while (!(RCC->CR & (1 << 17)));

// ② 配 PLL：HSE → ×9 → 72MHz
RCC->CFGR |= (1 << 16);             // PLL 源 = HSE
RCC->CFGR &= ~(1 << 17);            // 不分频
RCC->CFGR |= (0x7 << 18);           // ×9
RCC->CR |= (1 << 24);               // 开 PLL
while (!(RCC->CR & (1 << 25)));

// ③ 总线分频：APB1 ÷2 = 36MHz
RCC->CFGR |= (0x4 << 8);

// ④ 切换 SYSCLK 到 PLL
RCC->CFGR |= (0x2 << 0);
while ((RCC->CFGR & 0xC) != 0x8);
⚠️ 踩坑：忘了 FLASH->ACR → 灯不闪 → CPU 卡死。
顺序铁律：Flash 等待周期 → 开 HSE → 配 PLL → 分频 → 切换。

D8 · SysTick 24位硬件定时器

详见教程：W1D7-D8-tutorial.html

SysTick vs 旧 delay

delay(volatile...)systick_delay_ms()
精度靠猜精确毫秒
换芯片重调改 LOAD
CPU 用法空转硬件计数，以后能用中断

核心公式

72MHz → AHB/8 = 9MHz → 每秒 9,000,000 周期
1ms = 9,000 个周期 → SysTick->LOAD = 9000

完整函数

void systick_delay_ms(uint32_t ms)
{
    SysTick->CTRL &= ~(1 << 2);           // 时钟源 = AHB/8
    SysTick->LOAD = 9000;                 // 1ms = 9000 周期
    SysTick->CTRL |= 1;                   // 启动

    while (ms--)                           // 循环 ms 次
        while (!(SysTick->CTRL & (1 << 16)));  // 死等 1ms

    SysTick->CTRL &= ~1;                  // 关闭
}

COUNTFLAG（bit16）自动清零机制

读 SysTick->CTRL 时，硬件自动把 bit16 清 0。所以 while 条件读到 bit16=1 的瞬间，标志位就已经清了，VAL 也自动重载——下次 wait 直接开始计时。

🎯 Week 1 总套路

GPIO 操作万能三步

① 开时钟 → ② 配模式（CRL/CRH） → ③ 读写数据（BSRR/IDR）

位操作三咒语

操作咒语含义
置位（写 1）REG |= (1 << n)第 n 位变 1，其他不变
清零（写 0）REG &= ~(1 << n)第 n 位变 0，其他不变
清多位（写 0 到连续 k 位）REG &= ~(0xF << offset)从 offset 开始的 4 位清零

硬件调试三步

① 编译不过 → 看第一条错误（后面都是连锁反应）
② 编译过但不工作 → 检查时钟开了没
③ 灯不闪 → volatile 加了没
